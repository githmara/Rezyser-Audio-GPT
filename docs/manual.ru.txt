Руководство по аудиорежиссуре AI - Полное руководство (Версия 13.5.1 – Издательская версия)

Добро пожаловать в Гибридную Студию Записи! Этот набор инструментов позволяет автоматически писать, планировать, форматировать и переводить сценарии радиопостановок и аудиокниг. Он оптимизирован для работы с профессиональными синтезаторами речи и локальными движками экранных читалок (NVDA).

Набор состоит из главной студии (Режиссура), инструмента для переводов и акцентов (Полиглот), инструмента для построения структуры файла (Архитектор Аудиокниг) и — начиная с версии 13.0 — Менеджера Правил, в котором без Python можно добавить новый акцент, шифр или творческий режим (см. ШАГ 7b ниже). Все четыре инструмента доступны в одном окне программы, переключаемом кнопками на панели или горячими клавишами Ctrl+1..Ctrl+4.



ШАГ 1: Первое запуск (Кросс-платформенный)

Приложение полностью портативно и готово к работе на системе Windows. Просто зайдите в папку и запустите файл run.bat (двойным щелчком или создав ярлык на рабочем столе). Откроется кратковременное окно терминала (исчезает автоматически), а через некоторое время появится главное окно программы — Режиссер Audio GPT. Никакого браузера, никакого сервера в фоне. Когда закончите работу, просто закройте окно программы.


Важно: Папка runtime/ является частью только готовых пакетов для системы Windows, загружаемых с вкладки Releases на GitHub. Она содержит изолированную среду Python, готовую к использованию без установки. Если вы клонируете репозиторий, этой папки нет и не должно быть — вы используете тогда виртуальную среду venv, описанную в разделе для разработчиков ниже.

--- Для разработчиков (Windows — репозиторий Git) ---

1. Клонируйте репозиторий на свой диск.
2. Запустите файл setup_dev.bat. Скрипт автоматически создаст виртуальную среду Python (venv) и установит все необходимые зависимости из файла requirements.txt.
3. Запустите приложение командой python main.py или через файл run_dev.bat — скрипт автоматически обнаружит папку venv и запустит программу.

--- Для разработчиков и обычных пользователей (macOS / Linux) ---

1. Клонируйте репозиторий на свой диск.
2. Откройте терминал в папке проекта и дайте скриптам права на выполнение: chmod +x *.sh
3. Запустите скрипт настройки: ./setup_dev.sh — скрипт создаст среду venv и установит зависимости.
4. Запустите приложение: python main.py или ./run.sh


Что вы увидите после запуска: После запуска программы откроется окно с начальным экраном. На нем вы найдете краткое описание доступных инструментов и раздел "System Check". Если ключ API еще не настроен, вы увидите красное сообщение, информирующее о том, что модули на основе AI (Режиссура, Переводчик) не будут работать — бесплатные локальные функции (акценты, Архитектор) активны сразу. Настройка ключа описана в следующем шаге.


Автоматические обновления

Приложение проверяет при запуске, доступна ли на GitHub более новая версия — проверка выполняется в фоновом режиме и не блокирует интерфейс.

Если вы установили программу с помощью готового установщика (.exe): при обнаружении новой версии появится диалоговое окно с предложением загрузить её. После подтверждения запустится индикатор выполнения, а когда загрузка завершится, приложение закроется автоматически и новый установщик запустится самостоятельно — ваши проекты и ключ API останутся нетронутыми.

Если вы используете репозиторий Git (разработчики и пользователи macOS/Linux): вместо предложения загрузки вы увидите уведомление о новой версии с прямой ссылкой на страницу релиза на GitHub. Загрузите оттуда архив «Source code» или обновите репозиторий командой git pull.


ШАГ 2: Подключение API (Ключ OpenAI)

Чтобы искусственный интеллект мог работать, ему нужен доступ к серверам OpenAI (создателей ChatGPT).

1. Создайте аккаунт на platform.openai.com и пополните счет на вкладке Billing.
2. Сгенерируйте ключ на вкладке API Keys (начинается с sk-proj-). Внимание: ключ отображается только один раз — в момент создания. Если вы закроете окно без копирования, он пропадет безвозвратно, и вам придется его аннулировать (опция revoke secret key) и сгенерировать новый. Скопируйте его сразу, прежде чем что-либо нажимать.
3. На начальном экране программы нажмите кнопку "Сгенерировать файл golden_key.env и открыть текстовый редактор". Система автоматически создаст чистый файл конфигурации в нужном месте и откроет Блокнот с шаблоном, готовым к редактированию.
4. В открытом файле вы увидите строку OPENAI_API_KEY=ЗДЕСЬ_ВСТАВЬТЕ_СВОЙ_КЛЮЧ. Удалите текст-заполнитель и вставьте свой скопированный ключ непосредственно после знака равенства (без пробелов). Сохраните файл и перезапустите приложение.

Подсказка: После перезапуска приложение проверит файл и отобразит зеленое сообщение в разделе "System Check" — это означает, что все модули AI готовы к работе. В противном случае оно четко укажет, что не так, и позволит открыть текстовый редактор, чтобы быстро исправить ошибку.


ШАГ 3: Главная студия режиссуры

Это ваш командный центр для придумывания историй и написания сценариев. Перейдите к нему, нажав кнопку "🎬  Режиссер" на панели кнопок или выбрав Инструменты → Режиссер из меню (горячая клавиша: Ctrl+1).



ВАЖНО: Формат файлов проектов — возврат к .txt

В версии 7 файлы проектов сохранялись в формате Markdown (.md), например, сценарии/анестезия.md. Это был целенаправленный эксперимент — формат .md должен был стать психологическим барьером для нетехнических людей, отговаривая их от открытия и ручного редактирования файла в Блокноте (а значит, от случайного повреждения заголовков структуры).

Эксперимент не удался. AI часто нарушает одно из фундаментальных запретов (подробнее об этом позже). По этой причине в версии 8 вернулись к простым файлам .txt — более интуитивно понятным и читаемым для каждого. Например, проект под названием "анестезия" теперь сохранится как сценарии/анестезия.txt, а его резюме как сценарии/анестезия_резюме.txt. Это позволит каждому легче вносить режиссерские корректировки или сокращения текста на ходу.


⚠️ КРИТИЧЕСКОЕ ПРАВИЛО МОНТАЖЕРА (ПРОЧИТАЙТЕ ПЕРЕД НАЧАЛОМ!) ⚠️

Все ваши сгенерированные сценарии и книги сохраняются в обычных текстовых файлах (.txt), которые легко открыть в Блокноте.

Как режиссер, вы ИМЕЕТЕ ПОЛНОЕ ПРАВО (и даже обязанность) редактировать этот файл! Если искусственный интеллект заходит слишком далеко и, например, сам решает проблему героев в конце сгенерированного фрагмента, откройте файл, удалите последние строки и оставьте текст оборванным в самом напряженном моменте, прежде чем очистить память, заново загрузить файл для перезагрузки контекстного окна и нажать "Отправить в AI" для продолжения.

🔴 ОДНАКО - ЧТО ВАМ НЕЛЬЗЯ ТРОГАТЬ:

Никогда не изменяйте, не удаляйте и не редактируйте строки, содержащие структуру (например, Акт 1, Сцена 2, Глава 1, Пролог). Если вы нарушите эти заголовки:
1. Ваша история развалится в экранных читалках (NVDA).
2. Архитектор аудиокниги, а значит и Студия Eleven Labs, не сможет правильно разделить вашу книгу на фрагменты для чтения.
3. Внутренний Конвертер приложения не сгенерирует правильного оглавления в файле Word или на временной шкале в ElevenLabs.

Редактируйте диалоги, сокращайте действие, но оставьте структуру в покое!


Как использовать Режиссуру?

- Книга Мира (боковая панель слева): Здесь вы записываете правила вселенной. Каждый проект теперь имеет собственную Книгу, связанную с именем файла — если ваш файл истории, например, emilia_heist_audiobook.txt, Книга Мира сохранится как сценарии/emilia_heist_audiobook.md. Поэтому порядок важен: сначала введите имя файла проекта в поле "Имя файла проекта", а затем можете сохранять Книгу. Кнопка "💾 Сохранить Книгу навсегда" остается серой, пока не введено имя. Загрузка существующего проекта кнопкой "Загрузить историю" автоматически загрузит и соответствующий ему файл .md — вам не нужно делать ничего вручную. Внимание: Без заполненной Книги Мира приложение заблокирует отправку запросов к AI — это преднамеренная защита, которая оберегает вас от генерации текста без каких-либо правил.
- Долговременная Память (боковая панель слева): Второе поле на боковой панели. Здесь AI автоматически сохраняет резюме сюжета, когда память начинает переполняться. Вы также можете ввести сюда свои заметки вручную — модель воспримет их как "что я уже знаю" перед чтением текущей сцены.
- Режим Мозгового Штурма: Введите свободную идею, и AI предложит вам 3 варианта развития сюжета. Для каждого варианта он сразу сгенерирует готовый блок "--- НАБРОСОК ПОДСКАЗКИ ---". После выбора одного из них вставьте эту подсказку в Режим Сценария или Аудиокниги и внесите возможные замечания или информацию для модели. Память сюжета не изменяется во время Штурма — вы можете экспериментировать без риска!
- Режим Чернового Сценария: Вставляете сюда "Набросок Подсказки", сгенерированный в Мозговом Штурме. AI напишет длинный фрагмент с полными тегами [Speaker X: Имя - эмоция] для диалогов и эффектами [SFX: ...] для звуков. Текст автоматически добавляется в ваш файл (например, новая_сцена.txt) в папке сценарии/.
- Режим Традиционной Аудиокниги: Режим для писателей. AI генерирует чистую литературную прозу — никаких тегов, никакой фонетики, классические диалоги, вплетенные в повествование. Текст добавляется в файл так же, как в Режиме Сценария.
- Автоматическое создание персонажей: Скрипт запрограммирован так, чтобы автоматически создавать новые теги [Speaker], если сюжет требует нового персонажа. Подсказка: Если AI создает интересного персонажа, скопируйте его тег и вставьте в Книгу Мира, чтобы модель запомнила его навсегда!

Управление проектом — три кнопки действий:
- Загрузить историю: Загружает существующий файл проекта в память. Доступно только когда память пуста и введено имя файла.
- Очистить текущую (оставить Резюме): Очищает только текущую историю. Резюме в Долговременной Памяти и Книга Мира остаются сохраненными — идеально для продолжения длинной книги после генерации резюме.
- Закрыть Проект (Жесткий Сброс): Полностью очищает проект. Удаляет историю, резюме, Книгу Мира и обнуляет все счетчики. Используйте при смене проекта или новой истории.

Историческая заметка — почему режимы теперь защищены от случайного изменения:

В более ранних версиях приложения никто не следил за согласованностью творческого режима в рамках одного проекта. Это привело к некоторой тонкой творческой возможности, которую можно описать так: вы начинаете историю в Режиме Сценария, вставляете "Акт 1" и "Сцена 1" (приложение в восторге от вашей последовательности), генерируете несколько экранов напряжения с тегами [SFX: Скрип двери] и [Speaker 1: Марек - разочарованный]... а затем — потому что вдохновение пришло — нажимаете "Вставить разрез Глава 1". Приложение без колебаний добавляет заголовок "Глава 1" в файл, сразу после "Сцена 3". Теперь у вас в файле гибрид радиопостановки 50-х годов и социального романа. ElevenLabs смотрит на эту структуру, видит заголовок главы в середине акта и решает, что это, вероятно, ошибка в матрице. Содержание выглядит как произведение авангардного художника. Архитектор Аудиокниг, в свою очередь, исходит из предположения, что вы взрослый человек и просто обрабатывает это без слова — но также без каких-либо гарантий результата. Начиная с версии 12.0, интерфейс просто блокирует изменение режима записи, когда в памяти уже есть история. Мозговой Штурм всегда доступен — вы можете там задумываться без ограничений.

Шоураннерские Подсказки — AI как сценарист сериала (с версии 7, сохранено)

Это самое большое изменение, введенное в версии 7, сохраненное без изменений. Оно не видно в интерфейсе, но радикально изменяет качество генерируемого сюжета.

Шоураннер в телевизионной индустрии — это человек, ответственный за весь сериал — он следит за тем, чтобы сюжет никогда не шел к легкому решению, а постоянно сгущался, накапливал конфликты и держал зрителя в напряжении на протяжении многих сезонов. В версии 7 эта философия была встроена непосредственно в системные инструкции AI.

Директива Anti-Closure — состояние в версии 8

Несмотря на тестирование различных подходов, запрет Anti-Closure часто нарушается моделью, особенно в Режиме Традиционной Аудиокниги. AI имеет сильную тенденцию "закрывать" историю при каждой возможности — вплетает обобщающие фразы, создает ложные счастливые концы или завершает сюжетные линии в середине главы, как будто каждый фрагмент является последним. Эффект таков, что следующий фрагмент начинается с нуля, без естественной непрерывности.

Именно поэтому так важна описанная выше роль монтажера: если AI нарушает запрет и сам закрывает сцену, ваша задача как Режиссера — открыть файл .txt, удалить эти последние фразы и оставить текст прерванным в напряженный момент перед повторной загрузкой его в приложение.

Каждый режим имеет встроенную директиву Anti-Closure:

- Мозговой Штурм: AI действует как сценарист многосезонного сериала. Каждая из 3 предложенных опций ДОЛЖНА наращивать препятствия, вводить новые дилеммы, открытые линии и клиффхэнгеры. Абсолютный запрет на предложение опций, в которых герои примиряются навсегда, преодолевают основное препятствие без серьезных последствий или история движется к поспешному концу. Запрет на придумывание дешевых, магических локаций, нарушающих архитектуру мира.
- Режим Чернового Сценария: Абсолютный запрет на завершение текста обобщениями, моральными выводами или рамками. AI должен естественно прерывать действие в напряженный момент (клиффхэнгер), избегая заключений, чтобы следующий фрагмент мог плавно продолжить повествование.
- Режим Традиционной Аудиокниги: Абсолютный запрет на завершение текста патетическими обобщениями, закрытием сюжетных линий или предположением о конце истории (например, "Она была готова к новым вызовам", "Она поняла, что пора идти дальше"). Текст должен прерываться в середине текущего действия, напряжения или открытой мысли.

Исключение из правила: Если вы прямо попросите о завершении, финале или эпилоге, AI создаст удовлетворительное завершение истории — запрет не действует, когда вы сами об этом просите.

Управление Актами и Сценами (Режим Сценария, с версии 7, сохранено)

Режим Чернового Сценария имеет выделенную панель "✂️ Управление Структурой" с кнопками для построения структуры радиопостановки в реальном времени.

- Вставить Акт X: Вставляет заголовок акта (например, "Акт 2") И автоматически добавляет под ним "Сцена 1" — потому что каждый новый акт начинается с первой сцены. Счетчик актов перескакивает на следующий номер, а счетчик сцен сбрасывается на 2 (готов к следующей сцене в этом акте).
- Вставить Сцену X: Вставляет только заголовок сцены (например, "Сцена 3") в рамках текущего акта. Счетчик сцен увеличивается автоматически.

Интеллектуальная загрузка файла: Когда вы загружаете существующий файл сценария, приложение автоматически сканирует его и устанавливает счетчики актов и сцен на правильные номера. Счетчик сцен всегда рассчитывается в рамках последнего акта в файле.

Пролог и Эпилог: Кнопки "📜 Вставить Пролог" и "🏁 Вставить Эпилог" общие для обоих режимов и доступны в той же панели.

Архитектор Аудиокниг автоматически преобразует Акты в основные навигационные точки (Heading 1) для ElevenLabs и NVDA — Сцены станут жирным текстом, видимым в документе, но невидимым в содержании.

Интеллектуальное Управление Главами (Режим Аудиокниги, с версии 5, сохранено)

LLM генерирует максимум около 5000 символов на один запрос — этого недостаточно для полного главы. Поэтому вы работаете итеративно: генерируете последующие фрагменты кнопкой "Отправить в AI", пока не решите, что данная глава готова. Затем нажимаете "Вставить разрез главы" — приложение добавляет заголовок в файл и память, а счетчик глав автоматически перескакивает на следующий номер.

Кнопки на панели:

- Вставить разрез (Глава X): Вставляет заголовок (например, "Глава 3") в файл и память. Счетчик глав обновляется автоматически.
- Вставить Пролог: Работает только в начале, когда память пуста. Защита от случайной вставки Пролога в середине истории.
- Вставить Эпилог: Закрывает историю. После вставки Эпилога с содержанием приложение блокирует генерацию новых фрагментов — это намеренно, потому что после завершения истории больше нечего дописывать.

Интеллектуальная загрузка файла: Когда вы загружаете существующий файл, приложение автоматически сканирует его и устанавливает счетчик глав на правильный номер. Если файл имеет Пролог и три главы, счетчик покажет "Глава 4" — без ручного подсчета.

Аккумулятор Сюжета — как работает память AI

В память модели попадает только чистый текст повествования — то, что AI фактически написал. Инструкции, опции и планы исчезают после использования и не засоряют контекст. Эффект? Модель дольше сохраняет фокус, лучше запоминает акценты персонажей и реже "уходит" от правил Книги Мира.

Индикатор "Память Модели" на экране показывает, сколько места уже занимает ваша история:
- Зеленый (до 150 000 символов): Модель полностью сосредоточена, у вас безопасный буфер.
- Желтый (150 000 – 175 000 символов): Вы приближаетесь к границе. AI может начать сокращать высказывания и игнорировать правила. Начните медленно закрывать сюжетную линию.
- Красный (свыше 175 000 символов): Критическая перегрузка!

Бесконечная Память (с версии 6, сохранено)

Когда загорается красный сигнал тревоги, вам больше не нужно вручную копировать резюме. Введите в Мозговом Штурме команду, содержащую слово "резюме" или "подведение итогов" — AI создаст краткую капсулу знаний о всей предыдущей фабуле. Затем нажмите кнопку "Сохранить Резюме (навсегда)" — приложение автоматически создаст файл _резюме.txt в папке скрипты/ и защитит его от случайного перезаписи. После сохранения очистите память, и вы можете продолжать создание в текущем файле. Резюме вы загрузите в поле Долговременной Памяти перед следующей сессией — и можете продолжать историю бесконечно, без каких-либо ограничений по длине. Вам нужно только помнить, что если весь файл истории превышает 175000 символов, чтобы безопасно прервать работу и вернуться к ней позже, вы должны сгенерировать и сохранить Резюме истории. AI теперь также добавляет к нему несколько последних абзацев истории в момент написания резюме.

Автоматическое резюмирование: Когда память входит в состояние тревоги, Мозговой Штурм автоматически переключается в аварийный режим — вместо того, чтобы сразу генерировать опции, он сначала создает подробное резюме всей предыдущей фабулы и сохраняет его в тегах <РЕЗЮМЕ>. Приложение обнаруживает эти теги и автоматически переносит резюме в поле Долговременной Памяти. Никакой ручной работы!

Интеллектуальная загрузка с файлом резюме: Когда вы загружаете проект, который уже имеет сохраненный файл резюме (например, анестезия_резюме.txt), приложение автоматически загружает резюме в Долговременную Память и оставляет Текущую Память пустой. Благодаря этому вы не тратите токены на повторную загрузку всей истории — модель знает контекст из резюме и готова к бесконечному продолжению.

ШАГ 4: Постпродакшн в Режиссуре

После написания истории у вас есть инструмент постпродакшна, доступный внизу панели Режиссуры (появляется только в Режиме Аудиокниги).

Присвоение Заголовков Главам (Режим Аудиокниги)

AI проанализирует всю историю и предложит описательные, литературные заголовки для каждой главы (а также возможного Пролога и Эпилога). Модуль итеративно проходит по главам из сохраненного файла и присваивает им короткий заголовок, анализируя только начала каждой главы (до 6000 символов). Возвращает только пронумерованный список заголовков — без переписывания содержания книги.

Внимание: Модуль требует, чтобы файл уже был сохранен на диске (сгенерирован или загружен). Если файл не существует, приложение отобразит соответствующее сообщение.

ШАГ 5: Полиглот AI

Приложение для переводов и принудительного применения акцентов на синтезаторах экранных читалок. Перейдите к нему, нажав кнопку "🌍  Полиглот" на панели кнопок или выбрав Инструменты → Полиглот из меню (горячая клавиша: Ctrl+2).

Режим 1: Обычный Переводчик (Использует AI - Платный)

Идеально для прослушивания рассказов на красивом, правильном иностранном языке.

- Загрузите файл кнопкой "Загрузить файл в память" или используя кнопку "Обзор..." для выбора файла в системном окне.
- Выберите режим "Переводчик AI", введите целевой язык (например, "Финский").
- Скрипт переведет содержание. Если вы загрузите файл .html, он переведет только текст, видимый для читателя, и обновит код языка в теге html.

- Если вы загрузите обычный файл .txt или .md (например, сценарий, сгенерированный в Режиссуре), скрипт после перевода автоматически сохранит результат как файл .html с встроенным языковым тегом — готовый для чтения NVDA без дополнительной работы.
- Поддерживает также файлы .docx.

Интеллектуальное разделение на блоки: Длинные тексты автоматически делятся на блоки до 10 000 символов и переводятся последовательно с индикатором прогресса.

Защита от сбоев при сохранении прогресса (НОВИНКА в версии 9): Каждый переведенный блок немедленно сохраняется в скрытый временный файл в системной папке приложения. Если перевод прерывается из-за исчерпания лимита API, при следующем запуске приложение автоматически обнаружит этот файл, восстановит уже оплаченный прогресс и спокойно завершит работу с места прерывания — без ручного копирования. После успешного завершения временный файл автоматически удаляется.

Почему это важно? В версии 8 подход к возобновлению перевода имел критическую, тихую ошибку: если не переименовать файл с частичным переводом и вырезать переведенный фрагмент согласно инструкции, а затем сразу возобновить перевод остального — приложение открывало выходной файл в режиме перезаписи. Оплаченный прогресс безвозвратно и незаметно стирался, прежде чем вы успевали что-либо заметить. Версия 9 полностью устраняет эту ловушку: запись сохраняется после каждого блока, независимо от того, что вы сделаете потом.

Режим 2: Режим Режиссера / Акценты (Бесплатно - Работает локально)

Инструмент для "искажения" текста. Заставляет синтезаторы говорить на ломаном польском.

Интегрированное выпадающее меню Режима Режиссера позволяет выбрать акцент прямо из списка. Доступные опции:
Английский (Samantha / Mark — Vocalizer/Apple, Zira / Hazel — OneCore)
Русский (Milena — Vocalizer/Apple, Irina / Pavel / Yuri — OneCore, eSpeak)
Французский (Thomas / Amelie — Vocalizer/Apple, Julie — OneCore)
Немецкий (Markus / Stefan — Vocalizer/Apple, Hedda / Katja — OneCore)
Испанский (Jorge / Monica — Vocalizer/Apple, Helena — OneCore)
Исландский (Gudrun — OneCore, eSpeak; внимание: Vocalizer не поддерживает этот язык!)
Итальянский (Alice / Luca — Vocalizer/Apple, Elsa — OneCore)
Финский (Satu / Mikko — Vocalizer/Apple, Heidi — OneCore)
А также две опции очистки без акцента:
"Никакой (Очистка БЕЗ нормализации чисел)"
"Никакой (Очистка С нормализацией чисел)"
Опция с нормализацией чисел автоматически преобразует цифры в их словесное представление (например, 15 становится "пятнадцать") перед обработкой — благодаря этому NVDA читает числа плавно, а если текст имеет принудительный акцент, число также его получит, без исключений.
Вы, наверное, спросите, сможет ли синтезатор склонять такие нормализованные числа в зависимости от контекста? Ответ: нет. И это целенаправленная функция. Как известно: "польский язык сложный" — а иностранец, коверкающий нашу грамматику, только добавляет аутентичности ломаному акценту!

Исправитель Тегов (НОВИНКА в версии 9): В конце списка вы найдете дополнительную опцию — Исправитель Тегов. Он не применяет никаких фонетических правил или очистки. Делает одну вещь: вставляет в файл указанный вами двухбуквенный код языка ISO (например, "fi" для финского, "en" для английского). Это полезно, когда у вас уже есть готовый файл — например, ранее переведенный Переводчиком AI — но по какой-то причине он вышел с неправильным или отсутствующим языковым тегом. Вместо того чтобы переводить его заново, просто запускаете Исправитель с правильным кодом ISO и через мгновение у вас будет корректный файл.

Скрипт очищает режиссерские ремарки в скобках — удаляет все фрагменты текста, заключенные в круглые скобки (например, "(с вкраплениями вдохов)", "(шепотом)", "(иронично)"). ElevenLabs должен читать только диалоги и повествование, а не режиссерские указания, встроенные в текст.

Скрипт также очищает маркдаун-разметку — удаляет хэштеги заголовков (например, #, ##, ###), знаки равенства, используемые как подчеркивания заголовков, а также одиночные запятые, которые могут нарушать ритм чтения синтезатором.

- Защитный экран: Скрипт защищает все HTML-теги. Искажает только диалоги и повествование.
- Работает локально и ничего не стоит!

Режим 3: Шифровальщик (НОВИНКА в 13.0 — игры с текстом, бесплатно, локально)

Третий переключатель в Полиглоте, рядом с Переводчиком AI и Режимом Режиссера: «Режим Шифровальщика (игры с текстом, случайные искажения)". Запускает шесть локальных алгоритмов искажения текста — для развлечений, загадок, языковых головоломок. После выбора режима появляется выпадающий список алгоритмов, а для одного из них (Шифра Цезаря) дополнительное числовое поле для смещения.

Доступные алгоритмы (выберите в меню «Выберите алгоритм искажения"):

- Перевертыш Текста (Чтение наоборот с заглавной буквой) — каждое предложение переворачивается знак за знаком, с сохранением точки/восклицательного/вопросительного знака в конце и заглавной буквы в начале. Перед переворачиванием скрипт разворачивает популярные аббревиатуры (например, и т.д., и т.п., т.е., д-р и т.п.), чтобы результат после переворачивания все еще имел смысл — без этого «и т.д." после переворачивания превращается в абсурдное «.д.т и".

  Вдохновение: миссия из программы «The Traitors. Предатели". Когда ведущая без предупреждения показала участникам популярную польскую песню из мультфильма — просто воспроизведенную в обратном порядке — реакции были в стиле «какой это язык?", «это, наверное, армянский", «венгерский или финский". Собственно миссия заключалась в том, что одна команда получала в лесу текст песни, записанный наоборот, должна была спеть его в этой перевернутой форме, другая команда это записывала, воспроизводила с конца — и в теории восстанавливала оригинальную песню для отгадывания. Точно такое же забавное недоразумение фонетическое генерирует наш Перевертыш, когда вы запускаете обработанный файл через синтезатор TTS.

- Типогликемия (Перемешанный центр слов) — классический психолингвистический эффект: «Вы не можете прочитать текст в любом порядке букв, если первая и последняя буква на своем месте". Скрипт случайным образом перемешивает центр каждого слова длиннее 4 знаков, первая и последняя буква остаются нетронутыми. Отлично подходит для теста «сколько я на самом деле вижу, а сколько догадывается мой мозг".

- Сломанный Телефон / Змеиный диалект — каждое «с", «з" или двузнак «ш" случайным образом удлиняется в шипение длиной 4-8 знаков («сссссс", «зззззз", «сссссш"). Вдохновение: Snecko из игры «Slay the Spire". Эффект: текст звучит, как будто его прочитал змей с дефектом речи.

- Гласный (Все гудит на «О") — каждая гласная в тексте заменяется на «о", с сохранением польских смягчений. «Добрый день" → «Дьонь доборо". Чем длиннее текст, тем забавнее эффект (а экранный чтец читает это с полной уверенностью).

- Сильное Заикание / Поврежденный Робот — добавляет заикания в начале слов: «ко-ко-ко-компьютер", «пр-пр-пр-душ". Для коротких слов (менее 3 знаков) алгоритм их пропускает. Если вторая буква слова согласная, повторяет две первые буквы (более естественный эффект); если гласная — повторяет только первую.

- Шифр Цезаря (смещение алфавита PL) — классический шифр подстановки: каждая буква смещается на N позиций в польском алфавите (AĄBCĆ…ZŻŹ, 35 знаков). Значение смещения вводите в числовое поле «Смещение шифра Цезаря (0 = случайное, диапазон -35…35)". Если введете 0 — приложение случайно выберет смещение в диапазоне 1-34 и покажет его в сообщении после завершения (чтобы вы могли потом расшифровать). Знаки вне алфавита (цифры, пунктуация, пробелы) остаются без изменений.

Практическое применение: шифры сохраняют результат в том же формате, что и Режим Режиссера (.html с тегом lang или .docx с тегами XML), то есть NVDA прочитает их голосом соответствующего языка. Особенно Перевертыш и Гласный звучат в синтезаторе намного интереснее, чем когда вы читаете их глазами — потому что мозг при чтении глазами автоматически пытается «раскодировать" текст, а TTS читает его точно так, как записано.

Все шесть шифров также описаны более широким, доступным языком в файле docs/dictionaries.pl.txt в пакете приложения — вместе с советами, как добавить собственный шифр типа «замена букв" без знания программирования (например, школьный жаргон, тарабарщина).

Детекция языка

Перед тем как скрипт "испортит" текст, он проверяет, на каком языке он написан. Если обнаружит, что это не польский, выведет предупреждение. Фонетические правила адаптированы под польский язык — их применение к тексту на другом языке дало бы непредсказуемые результаты.

Акценты оптимизированы под локальные синтезаторы (NVDA)

Акценты были на 100% разработаны для локальных, бесплатных и платных синтезаторов экранных чтецов — Vocalizer и eSpeak в NVDA. Это гарантирует стабильность и предсказуемость эффекта на любом компьютере.

Почему не ElevenLabs? Тесты на более длинных документах показали, что фонетическое искажение текста в движках, таких как ElevenLabs, не навязывает предполагаемый акцент — движок просто "угадывает" язык и из него скорее делает акцент западно/южнославянский. Локальные синтезаторы NVDA в этом вопросе намного более послушны.

Специфика локальных синтезаторов (TTS)

Разные движки TTS ведут себя по-разному. Ниже ключевые различия, которые имеют прямое влияние на работу акцентов:

eSpeak (Open Source)
Легкий, бесплатный движок с огромной базой языков — поддерживает, например, исландский. Очень строг к кодировке символов, что делает его предсказуемым. Однако имеет фонетические недостатки: часто делит фонемы между языками, из-за чего, например, украинский звучит с русским налетом.

Nuance Vocalizer
Платный коммерческий движок, встроенный бесплатно в устройства Apple (VoiceOver). Предлагает очень естественное качество произношения. Важное ограничение: НЕ ПОДДЕРЖИВАЕТ все языки — НЕТ исландского голоса. Для финского доступна Satu, для итальянского Alice.

Как скачать Vocalizer на Windows?
Vocalizer не встроен в систему Windows — это платные дополнения к NVDA, распространяемые компанией Tiflotecnia. Установка состоит из двух этапов: загрузки драйвера и самих голосов.

- Покупка и загрузка драйвера: Перейдите на сайт tiflotecnia.com (форма покупки на английском языке), приобретите лицензию и перейдите в раздел "Downloads", чтобы скачать файл драйвера. Важное замечание для NVDA: Подстраница "Downloads" — это архитектурный кошмар — заголовки на английском, навигация на португальском, а вся страница имеет вшитый португальский языковой тег в коде! Даже если вы включите переводчик в браузере, он переведет содержимое, но обычно не удалит языковой тег из кода. В результате экранный чтец может попытаться читать англо-польскую страницу с очень ломаным португальским акцентом. Не обращайте на это внимания и временно отключите автоматическое переключение языка, затем просто переведите страницу и скачайте драйвер!
- Загрузка голосов: Самого драйвера недостаточно — после его установки NVDA по умолчанию покажет ошибку загрузки синтезатора. Чтобы это исправить, нужно отдельно скачать пакеты с голосами, которые вы собираетесь использовать (например, польская Ewa для плавного чтения системных контролей и финская Satu для акцентов в Полиглоте).
Примечание: Только не забудьте включить автоматическое переключение языка после закрытия подстраницы, чтобы потом не разочароваться!
- Быстрая установка: Скачанные файлы — это пакеты плагинов NVDA (имеют расширение .nvda-addon). Вам не нужно искать их установку через NVDA -> Инструменты -> Магазин дополнений. Если NVDA включен, просто "откройте" эти файлы обычным способом (нажав Enter). Windows может скрыть их расширение, но всплывающая подсказка правильно определит их как "Файл пакета плагина NVDA". Чтец сам спросит, хотите ли вы установить дополнение.
После установки драйвера и голосов и перезапуска чтеца, Vocalizer появится в списке синтезаторов в настройках NVDA и не должен показывать ошибок.

Microsoft OneCore
Бесплатные голоса, встроенные в Windows (Mobile Voices). Хороший компромисс между качеством и доступностью языков. Предлагает высокое качество и поддерживает исландский (Gudrun), финский (Heidi) и итальянский (Elsa). Примечание: OneCore изначально НЕ поддерживает автоматическое переключение языка на основе тегов lang (если только вы не используете встроенный в систему экранный чтец Narrator) — см. раздел ниже.

Как скачать голоса OneCore на Windows? Голоса OneCore бесплатны, но большинство языков не устанавливаются по умолчанию. Чтобы скачать недостающие языки, зайдите в Настройки Windows → Время и язык → Речь → Управление голосами (или "Добавить голоса"). Выберите язык из списка (например, исландский, финский, итальянский) и нажмите Скачать. После скачивания голос появится автоматически в NVDA — без какой-либо дополнительной настройки.

Apple (iPhone / iPad / Mac) — VoiceOver с Vocalizer
На устройствах Apple движок Vocalizer встроен нативно в операционную систему и поддерживает все языки без каких-либо дополнительных покупок. По умолчанию установлены компактные версии голосов, которые звучат несколько механически. Чтобы получить значительно лучшее, более естественное звучание, стоит скачать расширенные или премиум версии:
- На iPhone/iPad: Настройки → Доступность → VoiceOver → Речь → Голос → выберите язык (например, финский) → коснитесь имени голоса (например, Satu) → вы увидите опции "Расширенный" и "Премиум". Скачайте выбранную версию — она занимает больше места на диске, но разница в качестве заметна.
- На Mac: Системные настройки (или Настройки системы) → Доступность → VoiceOver → Откройте инструмент VoiceOver → Речь → Голос → нажмите "Настроить" рядом с выбранным голосом и скачайте расширенную или премиум версию.

Android — Vocalizer платный за каждый голос
На Android Vocalizer доступен как приложение в Google Play Store (ищите "Vocalizer TTS"). Примечание: каждый язык и голос покупается отдельно — нет одной лицензии на все. Перед покупкой вы можете протестировать голос в течение нескольких минут в демо-режиме. После покупки голос привязывается к аккаунту Google и может быть установлен на любом устройстве с тем же аккаунтом.

Ключевая функция для незрячих: Файлы с языковыми тегами

Это настоящая революция. Обработанные файлы теперь сохраняются как:

- .html с атрибутом lang (например, lang="fi" для финского) — NVDA читает этот тег и автоматически переключает голос на соответствующий языковой синтезатор. Никакой ручной настройки!
- .docx с сохранением форматирования — каждый фрагмент текста имеет вшитый языковой тег в XML документа, так что Word и NVDA знают, каким голосом читать каждый фрагмент.

Обработанные файлы всегда сохраняются как НОВЫЕ документы (например, акцент_Финский.html).

Автоматическое переключение языка в NVDA — что нужно знать

Чтобы NVDA действительно переключало голос на основе тега lang, должно быть включено настройка "Автоматически менять язык, когда поддерживается". Вы найдете его в Предпочтениях NVDA → Речь. По умолчанию оно включено при первом запуске чтеца — но стоит проверить, не отключил ли его кто-то.

Важное ограничение голосов OneCore: Голоса Microsoft OneCore изначально не поддерживают этот механизм с NVDA, а только с встроенным в систему Windows Narrator. Существует неофициальное дополнение к NVDA под названием "One Core Autolang", которое пытается это исправить — но его развитие было быстро прекращено. Причина проста: каждый год, с первой версией NVDA в данном году, появляются изменения, ломающие совместимость API (так называемые API Breaking Changes). Иногда достаточно обновить поле "last tested version" в манифесте дополнения, но иногда требуются реальные изменения в коде — а автор не успевает с обновлениями. Если вам важно автоматическое переключение языка, более безопасным выбором будут Vocalizer или eSpeak, которые поддерживают этот механизм нативно, при этом в первом случае вы также должны учитывать временную недоступность обновленного драйвера голосов Vocalizer для последней версии NVDA из-за описанной несколько предложений ранее. Тогда у вас будет выбор:
- Не обновлять NVDA — голоса точно будут работать, но у вас не будет доступа к новым потенциальным функциям, улучшениям или исправлениям ошибок.
- Обновить NVDA и принудительно включить дополнение путем перезаписи совместимости (введено относительно недавно, сработает, если до сих пор требовалось только обновление манифеста). В противном случае... ну что ж, было предупреждение, что вы делаете это на свой страх и риск.

ШАГ 6: Архитектор Аудиокниг

Инструмент для подготовки профессиональной структуры файла под экранные чтецы и ElevenLabs. Перейдите к нему, нажав кнопку "📄  Конвертер" на панели кнопок или выбрав Инструменты → Конвертер из меню (сокращение: Ctrl+3).

Как это работает?

1. Введите путь к исходному файлу в текстовом поле или нажмите кнопку "Обзор...", чтобы выбрать файл в системном диалоговом окне. Скрипт принимает как .txt, так и .docx, а также файлы .html и .htm.
2. Нажмите "Создать Архитектуру для ElevenLabs".
3. Скрипт просканирует каждую строку текста и заменит ключевые слова (Акт, Глава, Пролог, Эпилог, Вступление) на заголовки "Heading 1" в документе Word.
4. Сцены (строки, начинающиеся со слова "Сцена") превращаются в жирный текст — видимый в документе, но невидимый в оглавлении ElevenLabs.
5. Весь оставшийся текст (диалоги, описания) проходит без изменений.
6. Вы получите файл архитектура_[название_оригинала].docx — профессиональный Word, готовый к дальнейшей работе. Например, если ваш входной файл назывался znieczulica.txt, выходной файл будет называться архитектура_znieczulica.docx и появится в той же папке, что и оригинал.

Очистка по пути: Скрипт автоматически удаляет HTML-теги (если текст поступает из браузера или из локального файла .html, например, сгенерированного Полиглотом) и метки заголовков Markdown (например, ### или ####) перед обработкой. Вам не нужно очищать файл вручную.

Мощное преимущество для пользователей NVDA и ElevenLabs

Форматирование "Heading 1" для глав — это не только эстетика. В NVDA вы можете быстро перемещаться по заголовкам, нажав клавишу H (или 1 для заголовков первого уровня). Вместо того чтобы пролистывать сотни страниц, вы перескакиваете от главы к главе за доли секунды. Это меняет способ работы с длинными документами. В ElevenLabs вы уверены, что движок автоматически вставит разрывы и не будет зависать во время режиссирования аудиоклипов.

ШАГ 7: Как писать Книгу Мира (Подсказка Архитектора и Мини-курс Markdown)

Искусственный интеллект, питающий Главное Студия, лучше всего понимает команды, когда они отформатированы на языке разметки Markdown. Если вы дадите AI в Режиссере только свободную идею (например, "две сестры в загробном мире"), машина запаникует из-за отсутствия деталей и создаст банальную, предсказуемую историю, игнорируя логику.

Чтобы этого избежать, используйте внешнего чат-бота (например, бесплатного ChatGPT или Claude) в качестве вашего личного Архитектора. В версии 13.0 вам больше не нужно переписывать подсказку из инструкции — в панели Режиссера, сразу под кнопкой "💾 Сохранить Книгу навсегда", вы найдете кнопку "📖 Подсказка Архитектора для AI…". Нажмите ее, и откроется диалоговое окно с готовым текстом подсказки в поле "только для чтения" и кнопкой "Скопировать в буфер обмена". NVDA объявляет все содержимое сразу после открытия окна — достаточно нажать "Скопировать", вставить (Ctrl+V) в ChatGPT/Claude и заменить фрагмент "[ЗДЕСЬ ВВЕДИТЕ СВОЮ ИДЕЮ]" своим началом сюжета.

Почему так, а не через копирование из инструкции? Диалог в GUI имеет ключевое преимущество доступности: он нативный и читается без помех экранным чтецом, не требует переключения в текстовый редактор (где польские "хвостики" и кавычки могут неправильно копироваться) и работает одинаково на каждой операционной системе. Тот же шаблон — диалог с подсказкой для копирования — используется также в Менеджере Правил (см. ШАГ 7b ниже).

Как AI отвечает: В ответ вы получите текст в двух частях. Первая — это Книга Мира, заключенная в блок кода Markdown (метки ```markdown … ```). Скопируйте ТОЛЬКО содержимое блока кода (без самих меток ```) и вставьте в поле "Книга Мира" в Режиссере, а затем нажмите "💾 Сохранить Книгу навсегда". Вторая часть — это рекомендация, какой из творческих режимов (Сценарий или Аудиокнига) лучше всего подходит для вашей идеи — используйте ее как подсказку при выборе режима в RadioBox.

Мини-курс Markdown в Книге: один хэштег (#) обозначает основную секцию (например, «# Персонажи»), двойной хэштег (##) — это подсекция, тире (-) создают списки, а двойные звездочки (**текст**) выделяют абсолютные, нерушимые правила (например, «**Герой НИКОГДА не лжет**»). Для персонажа, говорящего на иностранном языке, добавляйте тег с акцентом, например, «[Speaker 1: Марек] - имеет французский акцент». Ключевое слово «акцент» НЕОБХОДИМО — движок приложения автоматически обнаруживает его и накладывает правильные фонетические правила на диалоги данного персонажа.

Совет: Не забывайте периодически обновлять свою Книгу Мира по мере добавления новых мест или важных персонажей в сюжет. Вы можете делать это вручную или с помощью внешнего чат-бота. Чем богаче фундамент, тем лучше результаты в Режиссуре!

ШАГ 7b: Менеджер Правил (Ctrl+4) — словари для лингвистов без Python

Версия 13.0 вводит четвертый инструмент на панели: Менеджер Правил. Открываете его клавишей Ctrl+4 или из меню Инструменты. Это визуальный проводник папки dictionaries/, где приложение хранит все фонетические правила (акценты), шифры и режимы Режиссера в файлах YAML (формат, понятный человеку, проще, чем JSON).

Зачем нужен YAML? До версии 12.0 все акценты, творческие режимы и фильтры были «зашиты» в коде Python. Добавление нового акцента требовало программиста. В версии 13.0 все это вынесено в файлы .yaml в папке dictionaries/ — каждый, кто умеет открыть Блокнот, может настроить существующий акцент («заменить sz на sh на sch») или добавить совершенно новый (шведский, датский, русский диалект).

Что вы найдете в Менеджере Правил:
- Дерево файлов — полный список всех файлов YAML в dictionaries/ с разделением на базовые языки (в настоящее время полностью реализованы: Polski, English, Suomi, Русский) и категории (акценты/, шифры/, режиссер/). Навигация стрелками как в Проводнике Windows, полная поддержка NVDA.
- Кнопка «📝  Открыть в текстовом редакторе» — открывает выделенный файл в системном Блокноте. Редактируете, сохраняете, нажимаете «🔄  Обновить дерево» и готово.
- Кнопка «📋  Дублировать выбранный файл» — самый быстрый путь к новому акценту: берете существующий файл (например, finski.yaml), задаете новое имя (например, шведский), Менеджер создает копию с комментарием «измените поля метка и описание» и открывает ее в редакторе.
- Кнопка «➕  Новый файл правил…» — мастер с выбором типа (акцент, шифр, режим Режиссера, новый базовый язык, шифр алгоритмический). Для обычных случаев (акцент, шифр типа «замена букв») мастер создает готовый к редактированию шаблон YAML. Для случаев, требующих программиста (новый базовый язык, алгоритм типа «перемешивание середины слов»), мастер генерирует запрос для чат-бота AI — вставляете в ChatGPT, получаете готовый файл YAML, сохраняете в dictionaries/ и готово.

Практическое применение: Хотите добавить шведский акцент? Откройте Менеджер Правил, выберите dictionaries/pl/акценты/finski.yaml, нажмите «Дублировать», назовите файл шведский, измените в нем поле метка на "Шведский (Альва / Оскар)", код iso на sv, измените несколько правил замены и сохраните. Вернитесь на Главную страницу, нажмите «🔄 Обновить акценты Режиссера из YAML» и перезапустите приложение. Новый акцент появится в меню Полиглота (в Режиме Режиссера) и — если добавите его в Книгу Мира персонажа («[Марек] - имеет шведский акцент») — также в автоматическом обнаружении акцентов Режиссера.

Где искать дальнейшую документацию папки dictionaries/:
- Файл `docs/dictionaries.pl.txt` (доступен в пакете для конечных пользователей с вкладки Releases) — краткое, простым языком описание каждой подпапки, 6 готовых шифров, 8 готовых акцентов и идеи, что можно сделать самостоятельно, без знания Python.
- Файл `dictionaries/README.md` (доступен только при клонировании репозитория с GitHub — пакеты выпусков фильтруют файлы .md) — подробная техническая документация формата YAML с примерами, списками полей, правилами pipeline, регулярными выражениями. Этот документ адресован лингвистам, знающим основы программирования, и разработчикам.

Важные советы для пользователей экранных чтецов (NVDA)

1. Сообщения об успехе и ошибках

Когда приложение завершает действие (например, сохраняет файл или сталкивается с ошибкой), появляется диалоговое окно с заголовком, таким как "Успех", "Предупреждение" или "Ошибка". NVDA сразу объявляет это окно и читает его содержимое. Вам не нужно ничего делать — диалоги полностью доступны с клавиатуры, а фокус автоматически попадает на кнопку "OK". Больше нет трюков с префиксом "Alert" — нативные диалоговые окна wxPython поддерживаются напрямую чтецом.

2. Важное предупреждение: тихая обработка в фоне

После нажатия кнопок, таких как "Отправить в AI" или "Запустить Обработку", программа сразу начинает обработку в фоне. На экране появляется индикатор прогресса (Gauge), но NVDA НЕ читает его значение в реальном времени — экранный чтец молчит во время работы. Не нажимайте кнопку повторно. Подождите, пока не появится диалоговое окно с результатом или сообщением об ошибке — NVDA объявит его автоматически.

3. Навигация между инструментами

Перемещение между инструментами (Режиссер, Полиглот, Конвертер, Менеджер Правил) происходит в одном окне программы с помощью четырех кнопок на панели инструментов или через меню Инструменты (Alt, затем N). Доступные клавиатурные сокращения: Ctrl+0 (Главная страница), Ctrl+1 (Режиссер), Ctrl+2 (Полиглот), Ctrl+3 (Конвертер), Ctrl+4 (Менеджер Правил — новинка в 13.0). После переключения инструмента фокус автоматически попадает на описание нового модуля — NVDA сразу начинает читать новый вид.

4. Порядок табуляции и названия виджетов

Каждый элемент интерфейса имеет читаемое русское название — NVDA прочитает его при фокусировке. Порядок табуляции логичен: от описания инструмента, через конфигурационные поля, до кнопок действий. Здесь нет сюрпризов — навигация клавиатурой работает так же, как в любой другой нативной программе Windows. Все меню и кнопки на русском языке, и NVDA прочитает их русским голосом без каких-либо дополнительных настроек.

Важные примечания к версии 9.0 / Changelog

Новые функции и изменения в версии 9

Устойчивое к сбоям сохранение прогресса перевода (ИЗМЕНЕНИЕ — исправление критической ошибки)
В версии 8 возобновление прерванного перевода было связано с тихой, разрушительной ловушкой: если пользователь следовал инструкциям (ручное вырезание переведенного фрагмента), а затем сразу возобновлял перевод остального, приложение открывало выходной файл в режиме перезаписи — безвозвратно и неосознанно удаляя весь оплаченный прогресс. В версии 9 весь механизм был перепроектирован. Каждый переведенный блок теперь немедленно сохраняется в временный файл (.jsonl) с присвоенным индексом. Возобновление полностью автоматическое — приложение само обнаруживает файл и восстанавливает прогресс без какого-либо вмешательства пользователя. После успешного завершения временный файл автоматически удаляется.

Исправитель Тегов в Режиме Режиссера (НОВИНКА)
В меню Режима Режиссера добавлена новая опция — Исправитель Тегов. Внедряет в файл только указанный пользователем двухбуквенный код языка ISO, без применения каких-либо фонетических правил или очистки. Спасательный инструмент на случай, если готовый файл вышел с ошибочным или отсутствующим языковым тегом.

Автоматическая конверсия .txt в .html в Режиме Переводчика (ИЗМЕНЕНИЕ)
Текстовые файлы (.txt и .md), переводимые Переводчиком AI, теперь автоматически сохраняются в формате .html с встроенным языковым тегом. Благодаря этому NVDA сразу читает файл нужным голосом — без каких-либо дополнительных шагов со стороны пользователя.

Интеллектуальное именование файла в Архитекторе (ИЗМЕНЕНИЕ)
Архитектор Аудиокниг больше не сохраняет итоговый файл всегда под одним и тем же именем архитектура_готова.docx. Теперь имя зависит от входного файла — например, для znieczulica.txt итоговый файл будет называться архитектура_znieczulica.docx. Файл появляется в той же папке, что и оригинал.

Функции, сохраненные без изменений с версии 8

Возврат к формату .txt для файлов проектов (ИЗМЕНЕНИЕ)
Файлы проектов и резюме теперь сохраняются в формате .txt вместо .md (Markdown). Эксперимент с форматом .md как психологическим барьером перед редактированием оказался неэффективным — AI часто нарушает правило anti-closure и часто лучше редактировать файл на ходу и затем перезагружать память, чтобы AI, по возможности, не получало скрытого разрешения на нарушение запрета. Формат .txt более интуитивен и понятен для каждого.

Правило Монтажера — новое предупреждение
В инструкции (и в будущих версиях интерфейса) добавлено явное предупреждение, объясняющее, что можно, а что нельзя редактировать в файлах проектов. Пользователь имеет полное право редактировать диалоги и содержание, но абсолютно не может изменять линии структуры (Акт, Сцена, Глава, Пролог, Эпилог) — их повреждение портит навигацию NVDA, разделение в ElevenLabs и оглавление в Word.

Обновленный Промпт Архитектора (ИЗМЕНЕНИЕ)
Промпт для внешнего чат-бота (ChatGPT/Claude) был перепроектирован. Вместо возврата сырого кода Markdown, AI теперь возвращает ответ в двух четко разделенных частях: Часть 1 — это Книга Мира, заключенная в блок кода Markdown (легко скопировать без меток), а Часть 2 — это Рекомендация Режима как обычный текст вне блока — советует, лучше ли использовать Режим Сырого Сценария или Традиционного Аудиокниги. Также изменено описание тегов персонажей: вместо "звуковых" теперь говорится об акценте и тоне голоса в теге, например, [Speaker 1: Имя] - шепелявит и французский акцент.

Объяснение состояния директивы Anti-Closure (ДОКУМЕНТАЦИЯ)
Инструкция теперь содержит честное объяснение, что несмотря на встроенные запреты Anti-Closure, модель все еще часто нарушает это правило, особенно в Режиме Аудиокниги. Вместо того чтобы скрывать эту проблему, версия 8 ставит вопрос прямо и возвращает ответственность за "ручное закрытие" обратно к Режиссеру — отсюда Правило Монтажера как официальное рабочее средство и возврат к обычному текстовому формату.

Функции, сохраненные без изменений с версии 7

Промпты Шоуранера — Директива Anti-Closure (Режиссура — все режимы)

Все три режима Режиссуры сохраняют встроенные директивы шоураннера. Мозговой Штурм действует как сценарист многосезонного сериала. Режим Чернового Сценария и Режим Аудиокниги запрещают завершать текст резюме, моралями и нарративными рамками — за исключением случаев, когда пользователь прямо просит о финале или эпилоге.

Управление Актами и Сценами в реальном времени (Режиссура — Режим Сценария)
Режим Чернового Сценария имеет панель с кнопками "Вставить Акт X" и "Вставить Сцену X". Счетчики обновляются автоматически. Загрузка существующего файла устанавливает оба счетчика на правильные номера.

Очистка скобочных ремарок режиссера (Полиглот — Режим Режиссера)
Скрипт удаляет все фрагменты текста, заключенные в круглые скобки (например, "(шепотом)", "(с вкраплениями вдохов)"). ElevenLabs читает только диалоги и наррацию.

Бесконечная Память и Автоматическое Резюмирование (Режиссура — Аккумулятор Сюжета)
Когда индикатор памяти входит в состояние красного сигнала, Мозговой Штурм автоматически генерирует резюме сюжета и сохраняет его в поле Долговременной Памяти. Нажатие "Сохранить Резюме (навсегда)" создает файл _резюме.txt. После очистки памяти и загрузки резюме можно продолжать историю без каких-либо ограничений по длине.

Архитектор Аудиокниг
Инструмент поддерживает ключевые слова Акт, Глава, Пролог, Эпилог и Вступление как заголовки первого уровня. Сцены превращаются в жирный текст. Скрипт автоматически очищает HTML-теги и метки Markdown перед обработкой.

Интеллектуальное Управление Главами (Режим Аудиокниги)
Посвященная панель с кнопками для вставки Пролога, Эпилогов и разрезов глав прямо из интерфейса. Счетчик глав обновляется автоматически. Загрузка существующего файла устанавливает счетчик на правильный номер без ручного подсчета.

Присвоение Названий Главам (Постпродакшн — Режим Аудиокниги)
Модуль проходит по главам из сохраненного файла и присваивает им короткое, литературное название, используя быстрый AI-модель, анализируя только начала каждой главы.

Удаленные функции (без изменений с версии 7)

Карта Разрезов (Режим Сценария) — заменена панелью управления Актами и Сценами в реальном времени.
Адаптация Сценария на Аудиокнигу (Режим Сценария) — удалена по финансовым причинам. Рабочий процесс генерировал три отдельные затраты: генерация сценария, потенциальное резюмирование при переполнении памяти, а затем повторная генерация всего как прозы аудиокниги. Рекомендуется писать сразу в Режиме Традиционной Аудиокниги с самого начала.

Ранее промпт для внешнего чат-бота для генерации Книги Мира существовал только в этой инструкции — пользователь должен был скопировать его из текстового файла (что на экранных чтецах бывает проблематично из-за польских знаков и кавычек). В версии 13.0 панель Режиссера имеет новую кнопку "📖 Промпт Архитектора для AI…" (прямо под "Сохранить Книгу навсегда"), которая открывает нативное диалоговое окно с полным текстом промпта в поле READONLY и отдельной кнопкой "Скопировать в буфер обмена". Шаблон A11y идентичен для диалога результатов мастера в Менеджере Правил — NVDA сразу читает все содержимое окна без дополнительной настройки. Полное описание в обновленном ШАГЕ 7.

- Русский акцент в Полиглоте (НОВИНКА — восьмой европейский язык в меню Режима Режиссера). В выпадающий список "Режим Режиссера / Акценты" добавлена опция "Русский (например, Milena / Irina / Yuri / Pavel)". В отличие от остальных акцентов, которые только "портят" польский текст на уровне латинских букв (например, "sz" → "sh", "w" → "v"), русский акцент выполняет полную транслитерацию польского текста на кириллицу — благодаря этому русскоязычные голоса синтезатора TTS (Vocalizer Milena, OneCore Irina / Pavel / Yuri, eSpeak "ru") могут "произносить" слова. Расширено отображение на йотированные гласные ("ja" → "я", "je" → "е", "jo" → "ё", "ju" → "ю"), польские смягчения si/ci/ni/zi + гласная (например, "ciocia" → "цёця"), а также русские специальные диграфы: "ść" / "szcz" → "щ", "sz" → "ш", "cz" → "ч", "rz" → "ж", "ch" → "х".

  Фонетическое примечание: Акцент не является академической русской транслитерацией (ISO 9, GOST, BGN/PCGN). Целью является исключительно звуковое уподобление польского текста русскому — для нужд синтеза речи, а не для серьезной транслитерации собственных имен. Для этой второй задачи используйте специализированные лингвистические инструменты.

  Код языка для выходного файла: "ru" (ISO 639-1). NVDA с включенной опцией "Автоматически менять язык, когда поддерживается" сразу переключится на русский голос.

- Примечание о "ź": В русском языке нет точного аналога польского "ź", поэтому этот знак нормализуется до "ж" — так же, как "ż". Часть севернорусских диалектов и белорусский произносят "ź" ближе к "зь", но с точки зрения синтезатора речи разница несущественна. Если лингвист решит, что предпочитает "зь", он может самостоятельно заменить это одно правило в файле `dictionaries/pl/akcenty/rosyjski.yaml` — движок загружает YAML-ы динамически, так что это не требует перекомпиляции приложения.

- Расширенная панель навигации и меню Инструменты (ИЗМЕНЕНИЕ). Панель теперь имеет четвертую кнопку — Менеджер Правил — а меню "Инструменты" поддерживает новый ярлык Ctrl+4. Предыдущие ярлыки (Ctrl+0 Главная страница, Ctrl+1 Режиссер, Ctrl+2 Полиглот, Ctrl+3 Конвертер) остаются без изменений.

- Кнопка "Обновить акценты Режиссера из YAML" на Главной странице (НОВИНКА). После добавления нового акцента через Менеджер Правил (или ручного редактирования файла YAML) нажатие этой кнопки перезагрузит обертки акцент_* в движке Полиглоты и Режиссера, чтобы новый акцент был виден в меню и в автоматическом обнаружении акцентов на основе тегов персонажей в Книге Мира. Полиглот обнаруживает YAML-ы автоматически (без обновления), но Режиссер требует регенерации оберток — обычно после обновления нужно еще раз запустить приложение.

Функции сохранены без изменений с версии 12.0

- Нативный GUI для настольных компьютеров на основе wxPython (без браузера, без локального сервера).
- Панель навигации с горячими клавишами (теперь Ctrl+0/1/2/3/4 — добавлено Ctrl+4 для Менеджера Правил).
- Полная оптимизация для экранных чтецов (NVDA, VoiceOver) — польские доступные названия, правильный порядок табуляции.
- Защита творческого режима от случайного изменения во время работы.
- Жесткий Сброс проекта одним кликом в Режиссере.
- Автоматическая загрузка творческого режима из файла метаданных проекта.
- Все акценты из предыдущих версий (Английский, Французский, Немецкий, Испанский, Итальянский, Финский, Исландский) и Исправитель Тегов.
- Полные функции AI: Мозговой Штурм, Сценарий, Аудиокнига, Аккумулятор Сюжета, Бесконечная Память, Присвоение Названий.
